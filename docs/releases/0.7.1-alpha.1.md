# NoirOS 0.7.1-alpha.1 (alpha)

Data: 2025-10-10  \
Canal: alpha  \
Commit base: f2e06f4

## Correções
- Autenticação do NoirFS reforçada: bloqueia tela de login enquanto a senha do volume estiver incorreta (repetição de prompt).
- Prevenção de perda de dados: nunca reformata se o dispositivo não estiver vazio; formatação apenas em mídia realmente vazia.
- Primeiro uso orientado: fluxo de criação/confirmacão de senha ao detectar mídia vazia.
- Visibilidade de saúde: verificação de acesso ao banco `/etc/users.db` antes do prompt de login de usuário.

## Notas
- Em ambientes RAMDisk (volátil), cada execução inicia com mídia vazia. O fluxo de primeiro uso solicitará criação da senha.
- Para simular persistência entre execuções, substitua o backend do bloco por um arquivo de disco emulado (fora do escopo desta release).

## Checklist pós-update
1. `make clean && make && make run`
2. Na primeira execução, defina e confirme a senha do volume.
3. Reinicie e autentique com a mesma senha do volume para liberar o login de usuário.
4. Verifique a linha `[db] USER_DB acessivel.` antes do prompt de credenciais.
