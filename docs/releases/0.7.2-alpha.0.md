# NoirOS 0.7.2-alpha.0 (alpha)

Data: 2025-12-11  \
Canal: alpha  \
Commit base: 5f08d60

## Correcoes
- Kernel principal agora monta explicitamente a particao 2 da MBR (quando presente) com blocos de 4096B, evitando montar discos inteiros e garantindo que o login apareca apos o reboot.
- Instalador NGIS valida espaco minimo para dados (>= ~16MiB) e ajusta o layout BOOT/DADOS para discos pequenos, eliminando erros genericos durante a formatacao.
- Validacao extra das entradas MBR (tipo/LBA/faixa) e do wrapper de offsets, evitando corrupcao de device e opcodes invalidos apos o reboot.
- Pilha do kernel/instalador ampliada para 16 KiB, reduzindo risco de overflow durante a criacao massiva de diretorios/arquivos.

## Melhorias
- Default do disco de desenvolvimento passa para 128MiB e o boot padrao no wizard e 32MiB para evitar layouts inviaveis.
- Alvo `make test` adiciona testes unitarios de host para wrappers de bloco (offset/chunk) e parsing do MBR.

## Checklist pos-update
1. `make clean && make` (ou `make iso` para gerar a ISO do instalador).
2. `make disk-img` ou `DISK_SIZE=256M make disk-img` para um disco com folga.
3. `make run-installer-iso` e realize a instalacao com senha do NoirFS; confirme criacao das particoes.
4. Reboot com `make run-disk` e valide que a tela de login aparece apos montar o volume.
5. Opcional: `make test` para validar wrappers e parsing em modo host.
